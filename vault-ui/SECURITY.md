# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please send an email to security@example.com with:

1. **Description**: Clear description of the vulnerability
2. **Impact**: Potential impact and attack scenarios
3. **Steps to Reproduce**: Detailed steps to reproduce the issue
4. **Proposed Fix**: If you have a suggested fix (optional)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Status Updates**: Regular updates on progress
- **Fix Timeline**: Depends on severity and complexity

### Severity Levels

- **Critical**: Immediate fix required (< 24 hours)
- **High**: Fix within 1 week
- **Medium**: Fix within 1 month
- **Low**: Fix in next release cycle

## Security Best Practices

### For Users

1. **Private Keys**: Never share your private keys
2. **Wallet Security**: Use hardware wallets for large amounts
3. **Verify Contracts**: Always verify contract addresses
4. **HTTPS**: Only access the app via HTTPS
5. **Updates**: Keep your browser and wallet updated

### For Developers

1. **Dependencies**: Regularly update dependencies
2. **Audits**: Security audits before major releases
3. **Testing**: Comprehensive security testing
4. **Code Review**: All code must be reviewed
5. **Secrets**: Never commit secrets to repository

## Smart Contract Security

## Bug Bounty Program

We value the security research community. If you find a vulnerability:

1. Report it privately (see above)
2. Allow us time to fix it
3. Eligible for bug bounty rewards

### Reward Ranges

- **Critical**: $5,000 - $25,000
- **High**: $1,000 - $5,000
- **Medium**: $500 - $1,000
- **Low**: $100 - $500

## Disclosure Policy

- **Coordinated Disclosure**: We prefer coordinated disclosure
- **Public Disclosure**: After fix is deployed and users have time to update
- **Credit**: Security researchers will be credited (if desired)
